# ShareCard constellation backdrop â€” design

**Date:** 2026-06-29
**Issue:** The corrupted share card exports as a solid red block with no visible glitch
treatment. Separately, both share variants (normal + corrupted) should carry the game's
star-field look instead of the current flat gradient.

## Context

`src/components/share/ShareCard.tsx` renders a fixed **380Ă—475** card that
`src/utils/shareExport.ts` rasterises with `html2canvas`. Two things break / fall short:

1. **The "solid red" bug.** The corrupted variant draws its treatment with a CSS
   `repeating-linear-gradient` scanline overlay plus an **inset `box-shadow`**
   ([ShareCard.tsx:147-151](../../../src/components/share/ShareCard.tsx)). `html2canvas`
   mis-renders repeating gradients (fills solid with the first stop) and ignores inset
   box-shadows â€” so the export becomes a flat red field with the glitch detail gone.
2. **Flat background.** Both variants use `cardStyle`'s stack of CSS `radial-gradient`s,
   which `html2canvas` also renders unreliably and which doesn't match the game's sky.

`html2canvas` *does* rasterise inline **SVG** faithfully (gradients, filters) â€” that's why
the game's `StarField` (an SVG) exports cleanly. So the fix and the visual upgrade are the
same job: move the share-card background to SVG.

## Decisions (confirmed with user)

- **Normal:** a star-field backdrop in the game's visual language (dark base, nebulae,
  seeded stars, a couple of constellation-like line clusters). A *representative* field is
  fine â€” no need to reuse the real `CONSTELLATIONS` data.
- **Corrupted:** the same field, **palette shifted to crimson**, plus a **frozen** glitch
  treatment: SVG scanlines, 2â€“3 static mosh bars, a red vignette, and the existing
  chromatic-split headline. **No cracks** (prior crack attempts looked poor). No watcher/eye.
- A still PNG cannot animate, so all "glitch" is a baked-in frozen frame.

## Component: `ShareBackdrop`

New file `src/components/share/ShareBackdrop.tsx`. A pure, self-contained inline SVG â€”
**no engine/React-context dependency** (the share card is rendered off-screen and rasterised;
it must be deterministic and side-effect-free).

```tsx
function ShareBackdrop({ corrupted }: { corrupted: boolean }): JSX.Element
```

- `viewBox="0 0 380 475"`, `preserveAspectRatio="xMidYMid slice"`, absolutely positioned to
  fill the card (`position:absolute; inset:0`), behind the content. The card keeps its
  `overflow:hidden` + `borderRadius:14`, so the SVG is clipped to the rounded card.
- **Stars** are generated by a local seeded RNG (mirror `StarField`'s `seededRandom`) so the
  field is deterministic and looks composed rather than noisy. Two tiers (dust + medium),
  fixed counts tuned for the small canvas.
- **Constellation flavour:** 2â€“3 small synthetic clusters â€” a handful of star points joined by
  faint hairlines. Generated from the same seeded RNG; not the game's `CONSTELLATIONS`.
- **No animation** (`<style>` twinkle keyframes, glows via `feGaussianBlur` are optional but
  must be export-safe; prefer plain opacity over filters where a filter risks html2canvas
  artifacts).

## Edits to `ShareCard.tsx`

- Render `<ShareBackdrop corrupted={corrupted} />` as the first child of the card (before
  `padStyle`).
- **Remove** the `cardCxOverlay` div and its style object (the source of the bug).
- **Remove** the `radial-gradient`/`linear-gradient` stack from `cardStyle.background`; the
  card keeps its border, radius, size, and `overflow:hidden`. Background colour falls to the
  SVG base (set a solid `#05070e` fallback so the card is never transparent before the SVG
  paints / if it fails).
- **Keep** `headlineCx` (chromatic-split headline) â€” it uses `text-shadow`, which exports.
- Row backgrounds (`ROW_BG`/`#0a1018`) and the fade mask are unchanged â€” they sit above the
  backdrop and stay legible over both palettes. Verify the corrupted nebula doesn't wash out
  row contrast; nudge row background opacity if needed.
